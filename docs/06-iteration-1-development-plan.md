# 第一迭代开发计划

## 1. 迭代概述

第一迭代目标是跑通 Enterprise AI Platform 的最小可用业务链路：

```text
前端登录
→ 网关转发
→ 用户身份认证
→ 权限校验
→ 创建 AI 会话
→ 发起 AI 问答
→ SSE 流式返回结果
→ 保存会话和消息记录
```

本迭代不追求完整的知识库和 Agent 能力，重点是建立可持续扩展的微服务工程基础。

------

## 2. 迭代最终成果

第一迭代完成后，应具备以下能力：

1. 用户可以通过用户名、密码和租户编码登录。
2. 登录成功后获得 JWT 访问令牌。
3. 网关能够校验访问令牌。
4. 下游服务能够获得用户和租户身份。
5. 系统能够完成基础 RBAC 权限校验。
6. 用户可以创建 AI 会话。
7. 用户可以向大模型发送问题。
8. AI 回答可以通过 SSE 流式展示。
9. 会话和消息可以保存到 MySQL。
10. 用户可以查看和删除自己的会话。
11. 前端可以查看接口请求、响应和错误信息。
12. 项目可以通过 Docker Compose 启动基础依赖。

## 3. 第一迭代服务范围

### 3.1 gateway-service

负责：

- 统一入口
- 路由转发
- JWT 校验
- 清除客户端伪造的内部请求头
- 传递用户身份信息
- 跨域配置
- 请求编号生成
- 统一网关异常响应

### 3.2 auth-service

负责：

- 用户登录
- 密码校验
- JWT 生成
- 用户退出
- 登录失败处理
- 当前认证信息封装

### 3.3 user-service

负责：

- 租户查询
- 用户查询
- 角色查询
- 权限查询
- 用户角色关系
- 角色权限关系
- 为认证服务提供用户数据

### 3.4 ai-service

负责：

- 创建会话
- 查询会话
- 删除会话
- 保存用户消息
- 调用大语言模型
- SSE 流式返回
- 保存 AI 回答
- 记录 Token 和调用耗时

### 3.5 frontend

负责：

- 登录页面
- 保存访问令牌
- Axios 请求封装
- 登录状态管理
- AI 会话列表
- AI 聊天页面
- SSE 流式内容展示
- Network 和 Console 错误排查

## 8. Commit 规范

提交信息采用：

```text
类型: 修改内容
```

常用类型：

| 类型       | 说明                 |
| ---------- | -------------------- |
| `feat`     | 新增功能             |
| `fix`      | 修复问题             |
| `docs`     | 修改文档             |
| `refactor` | 重构代码             |
| `test`     | 增加或修改测试       |
| `chore`    | 工程、构建或配置修改 |
| `perf`     | 性能优化             |

示例：

```text
chore: initialize backend modules
feat: implement user login query
feat: add JWT authentication
fix: prevent cross-tenant conversation access
test: add authentication integration tests
docs: update local deployment guide
```

一次提交只解决一个明确问题。

## 11. 学习方式

每个开发任务按照以下流程进行：

1. 先阅读需求和设计。
2. 理解这一功能解决什么问题。
3. 创建独立 Git 分支。
4. 编写最小可运行代码。
5. 启动并验证结果。
6. 查看日志和接口响应。
7. 编写测试。
8. 提交 Git Commit。
9. 推送到 GitHub。
10. 创建 Pull Request。
11. 进行代码评审。
12. 根据评审意见修改。
13. 合并到 `main`。
14. 总结面试讲解点。

学习过程中不要求提前背完所有知识，而是在项目中遇到什么，就结合真实代码学习什么。

# 第一迭代详细设计

## 1. 迭代目标

第一迭代的目标是跑通系统最基础的业务闭环：

1. 用户通过账号和密码登录系统。
2. 系统完成用户身份认证并返回访问令牌。
3. 用户携带访问令牌访问受保护接口。
4. 系统校验用户身份和权限。
5. 用户发起一次知识库问答。
6. AI 服务返回最小可用的问答结果。
7. 前端页面能够展示登录结果和问答结果。

本迭代重点不是完成全部业务功能，而是验证前端、网关、认证、用户服务和 AI 服务之间能够正常协作。

------

## 2. 本迭代服务范围

### 2.1 gateway-service

职责：

- 接收前端的所有请求。
- 根据请求路径转发到对应微服务。
- 校验访问令牌。
- 传递用户编号、租户编号等身份信息。
- 统一处理跨域、日志和异常响应。

### 2.2 auth-service

职责：

- 处理用户登录。
- 校验用户名和密码。
- 生成访问令牌。
- 提供退出登录接口。
- 提供当前登录用户信息查询接口。

### 2.3 user-service

职责：

- 管理用户基础信息。
- 管理角色和权限。
- 根据用户编号查询用户、角色和权限信息。
- 为认证服务提供用户数据。

### 2.4 ai-service

职责：

- 接收用户的问答请求。
- 调用大语言模型。
- 返回问答结果。
- 保存基础会话记录。
- 为后续接入 RAG、Agent 和 MCP 预留扩展接口。

### 2.5 frontend

职责：

- 提供登录页面。
- 保存登录后获得的访问令牌。
- 调用后端接口。
- 提供简单的 AI 问答页面。
- 展示接口成功结果和错误信息。

------

## 3. 第一迭代完整业务链路

### 3.1 用户登录链路

1. 用户在前端输入用户名和密码。
2. 前端调用网关的登录接口。
3. 网关将请求转发到认证服务。
4. 认证服务调用用户服务查询用户信息。
5. 认证服务校验密码。
6. 校验成功后生成访问令牌。
7. 前端保存访问令牌。
8. 后续请求在请求头中携带访问令牌。

### 3.2 权限校验链路

1. 用户访问受保护接口。
2. 前端在请求头中携带访问令牌。
3. 网关解析并校验访问令牌。
4. 网关提取用户编号、租户编号和角色信息。
5. 网关将身份信息传递给下游服务。
6. 下游服务根据权限要求决定是否允许访问。
7. 无权限时返回统一的权限错误信息。

### 3.3 AI 问答链路

1. 用户在前端输入问题。
2. 前端调用 AI 问答接口。
3. 网关校验用户身份。
4. 网关将请求转发到 AI 服务。
5. AI 服务调用大语言模型。
6. AI 服务接收模型返回的回答。
7. AI 服务保存基础会话记录。
8. AI 服务将回答返回前端。
9. 前端展示问答结果。

------

## 4. 本迭代暂不实现的功能

以下功能不在第一迭代中实现：

- 文档上传和解析。
- 文档切片。
- Embedding 向量生成。
- Milvus 向量检索。
- Elasticsearch 全文检索。
- 完整 RAG 检索增强问答。
- Agent 工具调用。
- MCP 服务接入。
- 多租户管理后台。
- Prompt 在线配置。
- 模型动态切换。
- Kafka 异步任务。
- Kubernetes 部署。

这些功能将在后续迭代中逐步加入，第一迭代只保证基础微服务链路能够正常运行。
